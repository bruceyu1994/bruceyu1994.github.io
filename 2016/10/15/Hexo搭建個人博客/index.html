warning: LF will be replaced by CRLF

原因：windows中的换行符为 CRLF， 而在Linux下的换行符为LF，所以hexo执行git add . 时出现提示

解决方案：

  1. 删除你hexo根目录下面的.deploy_git文件夹
  2. 执行 $ git config --global core.autocrlf false //禁用自动转换

  3. 重新执行如下命令，发现已没有之前的警告了

    $ hexo clean
    $ hexo d -g

  4. 打开博客网址，进行验证是否成功

    参考链接

注：

CRLF表示句尾使用回车换行两个字符(即我们常在Windows编程时使用”\r\n”换行)
LF表示表示句尾，只使用换行
CR表示只使用回车

Permission denied (publickey).

部署时执行 hexo d 命令时出现该错误

fatal: Could not read from remote repository.

【Reason】You’ve probably not added a public key to your SSH keys

生成 publickey ，命令：$ ssh-keygen -t rsa; 然后将 publickey 添加到远程服务器即可。
